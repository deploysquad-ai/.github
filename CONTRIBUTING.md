# Contributing to deploysquad-ai

Thanks for your interest in contributing! We welcome contributions of all kinds.

## Filing Issues

- Search existing issues before opening a new one
- Use the provided issue templates (bug report or feature request)
- Be specific — include reproduction steps, environment details, and expected vs actual behavior

## Submitting Pull Requests

1. Fork the repo and create a branch from `main`
2. Make your changes with clear, focused commits
3. Ensure tests pass before submitting
4. Fill out the PR template completely
5. Link the relevant issue if applicable

## Code Style

We follow the conventions of each project's language ecosystem. Check the individual repo's `CONTRIBUTING.md` for project-specific guidelines.

## Community

- Be kind and constructive — see our [Code of Conduct](CODE_OF_CONDUCT.md)
- Questions? Open a discussion or issue
